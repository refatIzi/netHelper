Як налаштувати параметри збереження подій

  1. Перейти в вікно Logging для налаштування правил збереження подій (System->Logging)

  1. Створити нове правило для збереження системних подій рівня “critical” та “error” на диск маршрутизатора. Встановити для таких подій префікс “SYS-ERR”.

  1. Створити нове правило для збереження системних подій рівня “info” на диск маршрутизатора. Встановити для таких подій префікс “SYS-INFO”.


Як обмежити доступ до локальних мереж тільки з MAC-адрес дозволених пристроїв

  1. Зареєструвати на DHCP сервері список дозволених пристроїв

-          для цього необхідно підключитись Winbox на маршрутизатор, який виконує роль DHCP-сервера, відкрити вікно налаштувань DHCP Server (IP->DHCP Server) та перейти на вкладку Leases.

-          зареєструвати MAC-адреси пристроїв та закріпити за ними постійні IP-адреси

  1. Налаштувати DHCP-сервер на видачу адрес тільки зі встановленого списку MAC-адрес/IP-адрес

-          в вікні налаштувань “DHCP Server” перейти на вкладку “DHCP”

-          для DHCP Server’а та відкрити вікно налаштувань для моста локальної мережі роутера;

-          у вікні налаштувань DHCP-сервера встановити для опції “Addresses” значення “static-only”;

-          в налаштуваннях DHCP Server встановити опцію “Add ARP For Leases”;

-          перейти у вікно налаштувань моста (Bridge, вкладка Bridge) та відкрити налаштування відповідного моста

-          в налаштуваннях bridge-интерфейса встановити для опції “ARP” значення “reply-only”.

УВАГА!!! Якщо в мережі використовуються вузли, які потребують наявності статичної адреси та мають бути доступні навіть в разі недоступності DHCP сервера рекомендовано такі вузли зареєструвати статичними записами в ARP-таблиці маршрутизатора

Для цього потрібно перейти в вікно ARP та додати відповідні записи в ARP-таблицю маршрутизатора та зробити її статичною.

Як обмежити доступ до сервісів управління маршрутизатором Mikrotik

  1. Відкрити вікно “IP Service List” (IP->Services).

  1. У вікні “IP Service List” почергово відключити всі сервіси, окрім сервісів “ssh” та “winbox”

або

  1. Обмежити доступ до сервісів “ssh” та “winbox” тільки з IP-адрес адміністраторів мереж.
